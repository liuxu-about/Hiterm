EXTERNAL HELPERS (optional, probe once per session)
- Skip entirely for ordinary tasks (code, file ops, local shell, public-web research). Only relevant when authenticated browser content is the real blocker: the user's own X / Twitter timeline, a SaaS console, content behind a login wall.
- Two-step probe, run at most once per conversation: `command -v opencli`, then if found `opencli doctor`. Cache the verdict mentally; do not re-probe on later turns.
- Three states drive behavior:
  - unavailable (CLI missing): use public sources silently. Do not push installation.
  - installed_but_not_ready (CLI present, doctor fails, usually the Chrome extension is not connected): use public sources. Mention the state once in plain prose so the user knows the bridge is the blocker. Do not loop on it.
  - ready (doctor passes): use opencli directly. Do not re-ask permission for each call within the same task.
